# Jerry's Coding Workflow

**Version 1.0.0**

A disciplined coding methodology that ensures every task is planned, researched, built minimally, tested incrementally, and verified fully.

> "寫 code 前先想清楚，能搜就搜不重造輪子，寫一個測一個，全部完成再全面驗證。"

---

## The Four Phases

| # | Phase | What You Do |
|---|-------|-------------|
| 1 | **Global Route Planning** | Plan architecture, data flow, files to touch, functions to create |
| 2 | **Ponytail + GitHub Search** | Search for existing code, apply ponytail ladder, build minimum |
| 3 | **Incremental Testing** | After each unit: syntax check, logic test, edge case verification |
| 4 | **Full Verification** | All endpoints, all features, no regression, no dead code |

## Why This Workflow

- **Prevents over-engineering** — ponytail mindset keeps code minimal
- **Prevents rework** — search before building means no wasted effort
- **Catches bugs early** — test each unit before building the next
- **Ensures delivery quality** — full verification before sign-off
